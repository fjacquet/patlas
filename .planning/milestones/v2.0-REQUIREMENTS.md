# Requirements — vatlas v2.0 (Offline-Capable, Redesigned, Better Deck)

**Milestone:** v2.0 · **Defined:** 2026-05-19 · Continues phase numbering from v1.0 (phases 12–17).

Derived from the approved design at `/Users/fjacquet/.claude/plans/i-need-to-add-delightful-kahan.md` and `Improvements.docx`. All v1.0 requirements are validated and archived in `milestones/v1.0-REQUIREMENTS.md`.

## v2.0 Requirements

### Privacy Governance (GOV)

- [x] **GOV-01**: ADR-0001 is amended with an "Accepted exception" section defining the bounded service-worker envelope (own injectManifest code, precache-only, no `runtimeCaching`, same-origin, mandatory privacy-guard import); PITFALLS Critical-2 is rewritten from "forbidden" to "permitted only under the ADR-0001 SW envelope".
- [x] **GOV-02**: `scripts/check-supply-chain.mjs` is narrowed to allowlist exactly `vite-plugin-pwa` + `workbox-*` (still denying arbitrary `service-worker*`) and strengthened to assert `src/sw.ts` exists and its first import is `./privacy/fetchGuard`; the gate is unit-tested for the passing case and each failure mode.

### Progressive Web App (PWA)

- [x] **PWA-01**: A user can install vatlas as a standalone app — a valid `manifest.webmanifest` ships with `scope`/`start_url` = `/vatlas/`, icons, and `display: standalone`.
- [x] **PWA-02**: A user can open and use vatlas with no network — a precache-only service worker serves the full app shell offline after first visit.
- [x] **PWA-03**: The service worker cannot exfiltrate data — `src/sw.ts`'s first statement imports `./privacy/fetchGuard`, there is no `runtimeCaching`, and only same-origin precached assets are served; workbook data never reaches the SW.
- [x] **PWA-04**: A pending update never silently destroys a loaded estate — when a workbook is loaded the user gets a reload toast; auto-reload happens only when the snapshot store is empty.

### Navigation (NAV)

- [x] **NAV-01**: Primary view navigation is a right-side vertical menu rendered before the drop zone (not the top bar), preserving the `role="group"` + `aria-pressed` idiom and arrow-key wraparound, in EN and FR.

### UX / Visual Redesign (UIX)

- [x] **UIX-01**: The global dashboard summary and operational insights render as grouped KPI tile sections (icon, uppercase label, large value, sub-caption, semantic color accent) matching the vsizer reference, in light and dark themes, with every value identical to v1.0.
- [x] **UIX-02**: The per-cluster columns and the cluster-detail drill use the same KPI tile system, preserving gauges, sparklines, and StretchedPill.
- [x] **UIX-03**: Icons are vendored from the local `icons` project as same-origin inline-SVG React components (`currentColor`, zero runtime dependency added to vatlas).

### Capacity Planning (PLN)

- [x] **PLN-01**: Changing the planned CPU/RAM ratio produces a visible factual result — a measured-vs-planned headroom visualization (estate + per-cluster) via the single `<Chart>` plus KPI tiles, built from already-computed `view.globals` vs `view.plannedView.globals`, with no new memo and no editorial verbs, in EN and FR.

### PPTX Quality (PPT)

- [x] **PPT-01**: Deck charts are crisp and correctly proportioned — each chart SVG is rendered at its slide-box aspect ratio and at print-grade resolution, not a fixed size then stretched.
- [x] **PPT-02**: The deck surfaces the previously-dropped estate facts (per-cluster physical capacity, min/max/mean ratios, capacity headroom, named threshold/flag context, storage & host detail, stretched-cluster reservation) as factual brand-free text in EN and FR with no editorial verbs.
- [x] **PPT-03**: Tests assert slide content presence (key facts in `a:t` runs), image pixel-dimension/aspect, EN/FR chart-label parity, and a golden re-render of the canary fixture.

## Future Requirements (deferred)

- Single-file portable `build:portable` (`vite-plugin-singlefile`) as an offline distribution form of the app itself — independent of the per-snapshot HTML report. (Endorsed in CLAUDE.md "Stack Patterns by Variant"; not in v2.0 scope.)

## Out of Scope (v2.0)

- **Cross-session persistence of dataset rows** — unchanged hard invariant. The v2.0 SW exception is precache-only for static assets and never touches dataset rows; `refresh = data gone` still holds.
- **Richer / branded PPTX graphical treatment** — the deck stays brand-free (PPT-02 design decision); v2.0 only fixes crispness/sizing and text density.
- **Telemetry / backend / cross-origin fetch** — unchanged hard invariants.
