## Spacing Scale

Declared values (multiples of 4; matches the shipped components verbatim — do not introduce new spacing tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | `gap-1` segmented-control inner gap; `p-0.5` toggle frame padding |
| sm | 8px | `gap-2` checkbox↔label, stat label↔value; `mb-2` sub-headings |
| md | 16px | `panel` padding (`p-4`); `gap-4` panel-internal section stack |
| lg | 24px | `gap-x-6` insight/stat-grid column gap; `gap-y-3` grid row gap |
| xl | 32px | `gap-8` before/after DR column pair gap |
| 2xl | 48px | Reserved — major Planning-screen section breaks (ratios block ↔ DR block) |
| 3xl | 64px | Reserved — page-level only; not used inside Planning panels |

Exceptions:
- Numeric input field min touch target: input height `h-9` (36px) is below 44px but is acceptable as it matches the shipped DR-toggle button height (`h-9`) and the preset-button height; preserve visual rhythm over a bespoke 44px exception. Preset buttons use `h-8` (32px) verbatim from the shipped `AllocationSliders` preset group — keep for consistency.
- Planning-screen two-block separation uses the `2xl` (48px) break between the planned-ratios block and the DR-sim block to enforce the D-03 structural "measured vs planned never share a tile" separation visually.

## Typography

Matches shipped vatlas type usage exactly (sizes are Tailwind defaults already in use across `DrSimPanel`/`OperationalInsights`/`AllocationSliders`; 2 weights only):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / caption | 12px (`text-xs`) | 400 (normal) | 1.5 |
| Label / stat / control | 14px (`text-sm`) | 600 (semibold) for labels, 400 for values | 1.5 |
| Panel heading | 20px (`text-xl`) | 600 (semibold) | 1.2 |
| Display (evacuee / headline figure) | 24px (`text-2xl`) | 600 (semibold) | 1.2 |

Rules:
- Exactly 2 weights: `font-normal` (400) and `font-semibold` (600). No other weights.
- Exactly 4 sizes: 12 / 14 / 20 / 24px. No new sizes for this phase.
- Every numeric value (ratios, GHz, cores, RAM MiB, verdict numbers) renders in `--font-mono` with `tabular-nums` (verbatim shipped idiom). Numbers are never pre-formatted in i18n strings; use the shipped `fmt*` helpers (`fmtRatio`, `fmtGhzValue`, `fmtInt`).

## Copywriting Contract

All strings land in BOTH `en/` and `fr/` namespaces (EN/FR parity is a hard constraint). No editorial verbs anywhere — the project denylist (`recommend` / `should` / `poor` / `good` / `better` / `worse-as-judgement` / high-med-low confidence words) is enforced. Numbers never embedded pre-formatted in strings (use `fmt*` + i18n interpolation count). Em-dash (`—`) is the "not derivable / not determinable" sentinel, verbatim with the shipped sentinel idiom.

| Element | Copy (EN — FR parity required) |
|---------|------|
| Primary control: ratio presets | CPU preset buttons: `1:1`, `4:1`, `8:1`, `VDI 10:1`. RAM preset buttons follow the same set/pattern. Buttons are factual numeric labels (no verbs). Default state: CPU `4:1`, RAM `1:1` pre-filled into the editable field. |
| Primary control: editable field | Field labels: "Planned CPU ratio (vCPU : pCPU)" / "Planned RAM ratio". Helper caption (factual echo, verbatim-pattern with shipped `AllocationSliders` echo line): "Planned lens — applies {cpu} CPU / {ram} RAM. Not the measured value." |
| Section labelling (D-03 structural separation) | Planned block heading: "Capacity planning — what-if". Every value in this block carries an inline "planned" qualifier in its label. The realized read-only reference carries a "measured" qualifier and a one-line factual link/caption: "Measured ratio — calculated from this snapshot (see Dashboard › Operational Insights)." The two qualifiers are different words on different surfaces; never the same tile. |
| DR mode selector | Exactly two segments: "Server loss" and "Site loss". No third segment. |
| DR Server-loss selection | Individual host multi-select legend: "Simulate these hosts failed". Per-cluster quick control label: "Hosts failed in {cluster}" with a numeric stepper "{n} of {m}". Neutral, reversible. |

## Interaction Contract

- **Keyboard:** the 4-segment `ViewToggle` and the 2-segment DR mode selector both keep Tab-into-group + Arrow Left/Right/Up/Down move-and-activate with wraparound + Space/Enter activate (native `<button>`). The preset group reuses the same `<fieldset>` idiom. The editable numeric field is a native `<input type="number">` (keyboard-operable by construction; `aria-label` required, EN/FR).
- **Reversibility:** every DR failed-selection (host checkbox, per-cluster stepper, site pick) and every ratio change is an instantly-reversible in-memory input. Unchecking / re-typing / re-picking restores immediately. No confirmation, no destructive styling, no persistence (D-06/D-10/G3).
- **Measured vs planned separation (structural, not just wording):** the planned-ratios block and any measured reference are on different surfaces with different qualifier words and a `2xl` visual break; they NEVER share a tile or a number (D-03). The realized value is never overwritten or hidden by planning input.
- **Custom Failover** is one in-panel affordance that re-runs the same Server/Site sim with planned ratios — never a 3rd mode, never a separate failure primitive (D-11).
- **Single-memo invariant:** all planned-projection + reworked DR-result numbers flow through the existing `useEstateView` / `buildEstateView`. No new `useMemo`, no recompute in the component (presenters only).
- **Privacy:** no URL-hash codec, no localStorage of dataset rows or planned inputs; refresh = data gone (D-06). No network calls.

## Claude's-Discretion Recommendations (within locked intent)

**Open Item 1 — Site-loss when zero stretched clusters declared (D-08):**
RECOMMENDATION: **Always show the Site-loss segment; disable the site picker with a factual inline note** rather than hiding the segment. Rationale: hiding a top-level mode makes the two-mode model unstable across snapshots and hides a capability the user may not realize depends on declaring stretched clusters. The factual note ("Site loss needs at least one cluster you have declared stretched. None are declared in this snapshot.") teaches the dependency without judgement and stays consistent with the calc-from-real-data / no-hidden-features posture. The Server-loss segment remains fully usable. This is a contract decision for the executor: segment present, picker disabled-with-note when `declaredStretchedSites.length === 0`.

**Open Item 2 — Verdict enum word vs pure numbers (D-09):**
RECOMMENDATION: **Show the `Verdict` enum WORD alongside the physical numbers** (`{cluster} — {verdict} ({physical headroom numbers})`), not numbers-only. Rationale: `absorbs`/`tight`/`overflows` are factual state words (not editorial verbs — they are on the kept-by-design factual side of the denylist, same as the shipped `verdict.*` keys already in use), they reuse the shipped engine enum with zero new logic, and they make the before/after result scannable. Constraint held: the word is rendered in neutral text color only — NO color/traffic-light, NO accent — exactly as the shipped `DrSimPanel` already renders `t(\`verdict.${p.verdict}\`)`. The numbers carry the precision; the word carries the at-a-glance state. This is a contract decision for the executor: render enum word + numbers, neutral styling.
