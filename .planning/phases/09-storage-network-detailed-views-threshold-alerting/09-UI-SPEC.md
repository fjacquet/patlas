# Phase 9 — UI Design Contract

> Visual and interaction contract for Storage / Network / Detailed Views + Threshold Alerting.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Pre-populated almost entirely from 09-CONTEXT.md (D-01..D-11), 09-RESEARCH.md, and the
> shipped Midnight Executive design system (8 phases of precedent). No new design mechanism
> is introduced this phase — every visual reuses a verified shipped idiom.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn gate N/A — established 8-phase Midnight Executive system; Tailwind v4 `@theme` CSS-first, no `components.json`, no Radix) |
| Preset | not applicable |
| Component library | none (hand-rolled presenters on Tailwind v4 utility classes; `src/index.css` `@theme` + `@layer components`) |
| Icon library | none — no icon font/library. **No alert/warning/alarm icons** (D-04 factual-only). The only glyph in scope is the literal back arrow `←` (shipped `ClusterDetail` precedent) |

**Source of truth for tokens:** `src/index.css` `@theme` block. Do NOT invent palette values — every
color below is an existing CSS custom property already shipped and used P3–P8.

## Spacing Scale

Declared values (Tailwind v4 default 4px-base scale; all multiples of 4 — shipped P3–P8 verbatim):

| Token | Value | Tailwind | Usage in Phase 9 |
|-------|-------|----------|------------------|
| xs | 4px | `gap-1` `p-1` | Segmented-toggle inner gap/padding (`gap-1`, `p-0.5` = 2px is the one shipped sub-4 exception, see below), metric-row vertical rhythm |
| sm | 8px | `gap-2` `py-1.5` (6px) | Detail metric `Row` vertical padding (`py-1.5`), badge inner padding |
| md | 16px | `gap-4` `p-4` | `.panel` padding (shipped), default element spacing, view-shell section gaps |
| lg | 24px | `gap-6` | Storage/Network view column gaps, lens-toggle ↔ chart separation |
| xl | 32px | `p-8` `gap-x-10` (40px) | Detail-screen outer padding (`p-8`, shipped `ClusterDetail`), detail grid column gap (`gap-x-10`) |
| 2xl | 48px | `gap-12` | Major section breaks within a view shell |
| 3xl | 64px | — | Reserved; not expected in Phase 9 (screen-fit detail screens forbid large vertical gaps) |

Exceptions:

- **`p-0.5` (2px)** on the segmented-control wrapper — shipped `ViewToggle`/`AccountingModeToggle`/
  `PlannedRatiosControl` border padding. This is a locked existing idiom; reuse verbatim, do not "fix" to 4px.
- **`py-1.5` (6px)** detail metric-row padding — shipped `ClusterDetail.Row`; the screen-fit
  constraint requires this tighter rhythm so all rows fit one 16:9 area without internal scroll.
- Screen-fit detail screens: NO internal scroll (`overflow-hidden`); spacing is chosen so content
  fits one viewport / one PPTX slide. Prefer the tighter shipped values over the canonical scale
  when a screen would otherwise overflow.

## Typography

Shipped scale (Tailwind v4 defaults, used verbatim P3–P8). Exactly 3 sizes in active use + 1 display.
Exactly 2 weights: `font-normal` (400) and `font-semibold` (600). `font-medium` (500) appears only
in the shipped `.label` component class — treat as the label exception, do not add a 3rd general weight.

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Display (detail screen title) | 24px | 600 | 1.2 (`leading-tight` via default) | `text-2xl font-semibold` | `<h2>` cluster/datastore/VM/ESX detail title (shipped `ClusterDetail`) |
| Heading (view/section) | 20px | 600 | 1.3 | `text-xl font-semibold` | StorageView / NetworkView section `<h2>`, threshold-config `<h2>` (shipped `PlannedRatiosControl` heading) |
| Body / metric value | 14px | 600 | 1.5 | `text-sm font-mono font-semibold tabular-nums` (numeric) / `text-sm` (label) | Metric rows, table cells, lens-toggle/segment labels, threshold count strings |
| Caption / factual note | 12px | 400 | 1.5 | `text-[12px] font-normal` (or `text-xs`) | Factual captions, em-dash-sentinel notes, "shared across N clusters" line, degrade message, threshold echo line (shipped `PlannedRatiosControl.echo`) |

Rules:

- Metric/numeric values ALWAYS `font-mono tabular-nums` (shipped invariant — column alignment).
- No pre-formatted numbers in i18n strings; numbers injected via `fmt*` helpers from `@/utils/format`
  with the active locale (shipped invariant; EN/FR parity).
- Branded-unit values (`MiB`/`GiB`) render through the shipped formatters; never raw `* 1.048576`.

## Color

All values are existing `src/index.css` `@theme` custom properties. Every color utility MUST carry its
`dark:` twin (shipped non-negotiable invariant — checker/grep-gated across P3–P8).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | light `bg-slate-50` / dark `bg-surface-900` (body); panels light `bg-white` / dark `bg-surface-800` | View-shell background, detail-screen background, table body |
| Secondary (30%) | `.panel` = `border-slate-200 bg-white` / dark `border-surface-700 bg-surface-800`; segmented-control wrapper `bg-white`/`dark:bg-surface-900` | Cards, table containers, the threshold-config panel, lens-toggle wrapper, chart container panels |
| Accent (10%) | `--color-accent-500` (gold ≈ #F9B935) | **RESERVED — see explicit list below.** Text-on-accent is `text-surface-900` |
| Active filled control | `bg-primary-600 text-white` (navy) | The selected segment of the **storage-lens** toggle and the **threshold-preset** group (the `AccountingModeToggle`/`PlannedRatiosControl` active treatment — NOT accent) |
| Neutral flag treatment (threshold) | `bg-accent-500/15` row tint + `border-l-2 border-accent-500` left rule; badge = `bg-accent-500/15 text-slate-700 dark:text-slate-200` pill | Flagged partition/datastore rows and the count badge — **factual marker only, NOT a verdict** (gold is the project's "factual marker, never editorial flourish" token per `index.css` comment) |
| Destructive | none | **No destructive actions in Phase 9** (in-memory config edit is non-destructive & reversible; refresh restores defaults). Leave unused |

**Accent (gold) reserved for — explicit closed list (do not extend):**

1. The active **top-level `ViewToggle`** segment (shipped rule — `bg-accent-500 text-surface-900`).
   This now includes the two new `Storage` and `Network` segments by extension. No change to the rule.
2. The **threshold flag marker**: flagged-row left rule + faint row tint + count badge
   (`bg-accent-500/15` family). This is the *factual* use of gold per the `index.css` token comment
   ("Gold accent (factual marker, never an editorial flourish)").
3. The shipped single DR headline figure (P6) — out of Phase 9 scope, listed for completeness.

**Explicitly FORBIDDEN for Phase 9 (D-04, locked):**

- NO traffic-light / red-amber-green on threshold flags.
- `--color-util-low/-mid/-high` (green/orange/red) MUST NOT be used for threshold alerting.
  They remain reserved for the shipped utilization gauges only (status, not judgment) and do not
  appear in Storage/Network/detail/alert surfaces.
- NO `--color-util-high` (red) row highlight, NO alarm/warning icon, NO "critical"/verdict color.
- Treemap & stacked-bar fills: derive from the **navy `--color-primary-*` scale** (sequential
  shade ramp by value) — NOT the util traffic-light palette, NOT verdict colors. Treemap node
  color carries magnitude only, never a pass/fail semantic (D-04 spirit; XSS note: pass datastore/
  portgroup names as plain `name` data, never an HTML `formatter`).

## Layout & Interaction Contracts (Phase-9 specific — prescriptive)

> These extend the template; the planner/executor treat them as the visual source of truth.
> Every item below has a shipped precedent cited from 09-RESEARCH.md.

### LC-1 — ViewToggle extension (D-05)

- Extend `src/components/ViewToggle.tsx` `AppView` union + `VIEWS` array IN PLACE with
  `'storage'` then `'network'` (order: `dashboard·inventory·hosts·planning·eos·trends·storage·network`).
- NO new nav component. Idiom unchanged: `<fieldset role="group">` + `<legend className="sr-only">`
  - `map(button aria-pressed)`, Arrow Left/Right/Up/Down wraparound, `focus-visible:ring-2
  ring-primary-500`, active segment `bg-accent-500 text-surface-900` (the existing rule — gold).
- i18n: add `nav.storage` / `nav.network` to the **`inventory`** namespace (BOTH `en/` + `fr/`) —
  ViewToggle reads `nav.*` from `inventory` (verified, line 28). No editorial verbs.

### LC-2 — Storage view shell IA (D-07/D-08)

- Top row: a Storage-by-X scope control (Cluster · ESX · VM · Datastore) AND the storage-**lens**
  toggle, both as the shipped `<fieldset role="group"> + aria-pressed` segmented idiom (LC-5).
- Lens A (consumption: provisioned vs in-use incl `.vswp`+snapshots): primary visual =
  **ECharts treemap** via the single `<Chart>` SVG site (register `TreemapChart` in `Chart.tsx`
  `echarts.use([...])` — one-line add). `label.show` + `upperLabel.show` + `breadcrumb.show`;
  cluster→datastore drill via treemap built-in breadcrumb. Fills = navy `--color-primary-*` ramp.
- Lens B (capacity vs used vs free): **stacked-bar** reusing the already-registered `BarChart`
  (`series:[{type:'bar',stack:'cap',name:'used'},{...,name:'free'}]`). Same navy ramp; no verdict color.
- Below the chart: the P3 `DataTable` + `ColumnPicker` + CSV-of-filtered (reuse verbatim). New
  column ids REQUIRE `inventory:col.<id>` keys in BOTH locale files (Pitfall 3); `<thead>` MUST use
  the same `flex w-full` + per-cell `flex-1` layout as the virtualized `<tbody>`.
- Rows/values not derivable (e.g. 42 unrelinkable blank-cluster datastores) → em-dash sentinel
  `—`, never fabricated onto a cluster.

### LC-3 — Network view shell IA (D-11)

- Sections: vSwitch (standard) table, dvSwitch (distributed) + dvPort table, vNetwork
  (VM→portgroup/VLAN) table — all via the P3 `DataTable` idiom (same `inventory:col.<id>` rule).
- **Optional-sheet factual degrade:** when `vNetwork`/`vSwitch`/`dvSwitch` are absent (the real
  8-sheet trimmed export), render a single factual line — i18n `network:empty.unavailable`
  ("Network inventory not available in this export.") in the shipped caption style
  (`text-[12px]`/`text-xs` `text-slate-500 dark:text-slate-400`). NO error styling, NO icon,
  NO crash, NO editorial verb. Mirrors the shipped `vDatastore`/`vPartition` empty pattern.

### LC-4 — Three click-drill detail screens (D-06, the P5 `ClusterDetail` precedent)

- Pattern: lifted in-app view-state (NOT a router, NOT a 2nd `useMemo`). Drill state lives in the
  Storage/Network view shell (the way `GlobalDashboard` owns the cluster-detail selection), not in
  `App.tsx`. Props shape: `{ detail, onBack }` (copy `ClusterDetail` verbatim).
- **Screen-fit / export-ready contract (P5 "one screen fit = one PPTX slide", extended):**
  `<main className="flex-1 overflow-hidden p-8">`, inner `flex h-full flex-col gap-4`, a fixed
  `grid` (`grid-cols-1 md:grid-cols-2 lg:grid-cols-3`, `gap-x-10`), NO internal scroll, sized for a
  16:9 area. Phase 10 snapshots each as exactly one slide.
- Header row: `<h2 text-2xl font-semibold>` title (break-words) + a right-aligned **back affordance**
  — `← {t('detail.back')}` text button, `text-primary-600 dark:text-primary-400`,
  `hover:bg-slate-100 dark:hover:bg-surface-800`, `focus-visible:ring-2 ring-primary-500`
  (shipped `ClusterDetail` button verbatim).
- Metric rows: the shipped `Row` component shape — `flex items-baseline justify-between`,
  label `text-sm text-slate-600 dark:text-slate-400`, value
  `font-mono text-sm font-semibold tabular-nums`, `border-b border-slate-100 dark:border-surface-800`.
- **Datastore detail:** capacity / provisioned / in-use / free, VMs-on-it (sourced from the
  `vInfo.Path` map built for the relink — single source), host count, threshold flag marker if over.
- **VM detail:** vCPU/vRAM, disks, partitions (each partition row carries the threshold flag
  marker when `consumedMib/capacityMib ≥ threshold`), portgroups/switches, datastores.
- **ESX detail:** AUGMENTS the shipped `src/components/hosts/` Hosts view (does NOT duplicate the
  P5 cluster-detail drill). Datastores mounted + vSwitch/dvSwitch/uplinks per host.
- Drill entry points: a row/card click in the Storage view (datastore→Datastore detail,
  VM→VM detail) and within the Hosts view (host→ESX storage+network detail). Clickable rows get
  `cursor-pointer hover:bg-slate-50 dark:hover:bg-surface-800` + are keyboard-activatable
  (native `<button>` or `role` + Enter/Space — match the shipped clickable-card pattern).

### LC-5 — Storage-lens toggle (D-07, the `AccountingModeToggle` idiom)

- Reuse `AccountingModeToggle`'s structure verbatim; change the union to
  `'consumption' | 'capacity'` and the i18n namespace to `storage`. Active segment
  `bg-primary-600 text-white` (NOT accent — accent stays reserved per §Color). Keep the literal
  `role="group"` + the `biome-ignore` comment (CI grep gate asserts the literal). Arrow-key wrap,
  `focus-visible:ring-2 ring-primary-500`.

### LC-6 — Threshold flag visual treatment (D-04, FACTUAL ONLY — locked)

- Flagged table row: faint tint `bg-accent-500/15` + left rule `border-l-2 border-accent-500`
  (carries both dark+light; gold = factual marker per `index.css`). NO red, NO icon, NO verdict text.
- Count badge (e.g. on a section header / Datastore detail): pill
  `inline-flex items-center rounded px-2 py-0.5 text-xs font-semibold bg-accent-500/15
  text-slate-700 dark:text-slate-200`, label text from i18n with the count injected via `fmt*`
  (no pre-formatted number, no editorial verb): e.g. `alerts:fs.count` →
  `"{count} partitions ≥ {pct}% used"` / FR parity `"{count} partitions ≥ {pct} % utilisés"`.
- The flag describes a measured fact against the user's own threshold; it never says
  poor/critical/warning/should/recommend (denylist). Doc-comments must not spell forbidden tokens
  literally (grep-gate gotcha).

### LC-7 — Threshold config control placement & interaction (D-01..D-03, the `PlannedRatiosControl` precedent)

- A single `<section className="panel flex flex-col gap-4">` titled via `<h2 text-xl font-semibold>`
  living **within the Storage view shell** (top, above the table — co-located with the data it
  governs; also surfaced compactly on the Network/detail surfaces only if the same slice value is
  shown — one control, one source of truth, no duplicate editors).
- Three editable native `<input type="number">` fields (filesystem used %, datastore used %,
  LU used % — LU default chosen by Claude in PLAN), each `h-9 w-20 rounded border font-mono
  tabular-nums focus-visible:ring-2 ring-primary-500` with the `dark:` twin set (verbatim
  `PlannedRatiosControl` input class). `min`/`max`/`step` attributes shared with an on-commit
  `safeNum` sanitizer (the shipped CR-01 guard pattern) — out-of-range/non-finite falls back to
  last valid value; **no error state, factual only**.
- Optional preset segmented group (`<fieldset role="group"> + aria-pressed`, active
  `bg-primary-600 text-white`) ONLY if the planner defines presets — defaults preset is the
  RVTools-Analyser default (fs ≥90% used, ds >85% used). Reuse `PlannedRatiosControl` shape verbatim.
- A `text-[12px] font-normal text-slate-500 dark:text-slate-400` factual echo line restating the
  active thresholds (the shipped `planned.echo` idiom), numbers via `fmt*`, no verb.
- State source: the in-memory Zustand `thresholds` slice in `src/store/snapshotStore.ts` ONLY
  (the `plannedRatios` precedent — REPLACE-never-mutate, no `localStorage`, no URL-hash; `clearAll`
  resets to defaults; refresh = defaults restored). NO new persistence surface.

### LC-8 — Charts (D-08, single `<Chart>` SVG site)

- Both treemap and stacked-bar render through the one sanctioned `src/components/Chart.tsx`
  (SVG renderer mandated; option built by a pure selector off the memoized `EstateView`).
- Container = a `.panel` (secondary surface). Watch `npm run check:bundle-size`
  (≤300 KB gz echarts chunk) after registering `TreemapChart` — verify the gate stays green.
- No raw-HTML `formatter` (XSS): datastore/portgroup names pass as plain `name`/`value` data.

## Copywriting Contract

All strings land in NEW i18n namespaces `storage`, `network`, `alerts` (keys in BOTH `en/` and
`fr/` — parity mandatory) except ViewToggle `nav.*` which goes in the existing `inventory` namespace.
No pre-formatted numbers in strings (inject via `fmt*`). No editorial verbs (denylist:
recommend/should/poor/good/critical/warning + traffic-light language). Strings below are the EN
contract; the executor authors the FR parity key-for-key.

| Element | Copy (EN contract) |
|---------|------|
| Primary action — open Storage view | ViewToggle segment label `nav.storage` → `"Storage"` (FR `"Stockage"`) |
| Primary action — open Network view | ViewToggle segment label `nav.network` → `"Network"` (FR `"Réseau"`) |
| Primary action — drill to detail | Clickable row/card (no button label; the row IS the affordance). Back: `← {detail.back}` → `"Back"` (FR `"Retour"`) — shipped `ClusterDetail` key reused |
| Storage-lens toggle labels | `storage:lens.consumption` → `"Consumption"` ; `storage:lens.capacity` → `"Capacity"` (FR `"Consommation"` / `"Capacité"`) — factual nouns, no verb |
| Empty state — network sheets absent | heading: none (single line). Body: `network:empty.unavailable` → `"Network inventory not available in this export."` (FR `"Inventaire réseau non disponible dans cet export."`) — factual, no next-step instruction (the data is the user's; no action to offer) |
| Empty state — no dataset loaded | reuse the shipped app-level empty/upload state (out of Phase 9 scope; do not author a new one) |
| Not-derivable value | em-dash sentinel `—` (shipped `na` idiom) for any metric not computable (e.g. unrelinkable blank-cluster datastore cluster identity); the shipped factual caption `t('na')`-class string where a sentence is needed |
| Shared-LUN factual line | `storage:vsan.sharedAcross` → `"Shared across {count} clusters"` (FR `"Partagé entre {count} clusters"`) — excluded from single-cluster rollups, no allocation guess (D-10) |
| Threshold flag count (filesystem) | `alerts:fs.count` → `"{count} partitions ≥ {pct}% used"` (FR `"{count} partitions ≥ {pct} % utilisés"`) |
| Threshold flag count (datastore) | `alerts:ds.count` → `"{count} datastores > {pct}% used"` (FR `"{count} datastores > {pct} % utilisés"`) |
| Threshold config heading | `alerts:config.heading` → `"Thresholds"` (FR `"Seuils"`) |
| Threshold config echo line | `alerts:config.echo` → `"Filesystem ≥ {fs}% · Datastore > {ds}% · LU > {lu}%"` (FR parity, separators kept, numbers via `fmt*`) |
| Error state | None. Phase 9 has no error surface: invalid threshold input silently falls back to last valid value (factual, `safeNum`); absent sheets degrade factually (above); a malformed workbook is caught by the shipped parser-error path (out of Phase 9 scope) |
| Destructive confirmation | None — no destructive action in Phase 9 (in-memory config edits are non-destructive and reversible by refresh) |
