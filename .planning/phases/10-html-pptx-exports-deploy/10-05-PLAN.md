<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main thread → export worker | store inputs / resolved strings cross into the worker |
| worker → network | any fetch in the worker throws by design (privacy invariant) |
| generated Blob → local download | a file is written to the user's disk via an object URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-17 | Information disclosure | font/asset fetch inside the export worker | mitigate | `fetchGuard` is the FIRST import in export.worker.ts (Task 1 grep-asserted); all assets inlined (plan 03); zero runtime fetch |
| T-10-18 | Information disclosure | worker error object leaking parsed workbook rows | mitigate | hand-built `{name,message}` only; never spread `err`/`cause`/rows (mirrors parser.worker.ts; T-04-04) |
| T-10-19 | Tampering | path traversal via a crafted vCenter label in the filename | mitigate | D-05 sanitize regex `[^A-Za-z0-9_-]→_` + `.slice(0,64)` + `||'estate'` fallback (Task 1 test asserts) |
| T-10-20 | Repudiation | PPTX silently blank in PowerPoint while structural tests pass | mitigate | Task 4 human-verify opens the real deck in PowerPoint (the only control for Pitfall 1) |
| T-10-21 | Tampering | hook imported into the worker breaking the privacy/purity contract | mitigate | Task 1 grep gate: no `useEstateView`/`useSnapshotStore` in export.worker.ts (Pitfall 2) |
</threat_model>
