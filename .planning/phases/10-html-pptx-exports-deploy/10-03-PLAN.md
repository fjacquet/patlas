<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-derived names → exported HTML | malicious VM/datastore/cluster name could attempt markup/script injection |
| chart SVG → report DOM | SVG comes only from ECharts SSR (trusted), never user strings |
| exported file → recipient's browser | the artifact is opened on an untrusted machine, offline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-09 | Tampering | stored XSS via malicious cluster/VM name in exported HTML | mitigate | Task 1 routes all user strings through React JSX text nodes (renderToStaticMarkup auto-escapes); the raw-markup-injection escape-hatch prop is forbidden for data and grep-asserted absent; explicit script-tag-name test |
| T-10-10 | Information disclosure | external font/asset ref leaking which machine opened the report | mitigate | Task 2/3: zero `<link>`/network ref; system-font default (0 bytes); CSP meta `default-src 'none'`; grep asserts no `http(s)://` |
| T-10-11 | Tampering | DOM-clobbering via duplicate anchor ids from repeated real names | mitigate | Task 1 deterministic `c-${i}-${slug}` ids; test on a 2-same-named-cluster fixture asserts no duplicate id |
| T-10-12 | Denial of service | oversized report (memory/open failure on recipient) | mitigate | Task 2/3 `assertSizeBudget` hard ceiling enforced by a 10k-fixture test (HTM-03) |
</threat_model>
