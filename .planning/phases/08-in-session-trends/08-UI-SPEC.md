## Spacing Scale

vatlas uses Tailwind's native 4px scale. Declared values (multiples of 4):

| Token | Value | Usage in Phase 8 |
|-------|-------|------------------|
| xs | 4px | `gap-1` segment gaps inside the extended `ViewToggle`; inline number/sign spacing in delta rows |
| sm | 8px | `gap-2` snapshot list rows; sparkline internal grid inset (`grid:{top:2,bottom:2,left:2,right:2}` ECharts px — chart-internal, not a layout token) |
| md | 16px | `p-4` panel padding (the shipped `.panel` component idiom) |
| lg | 24px | `gap-6` between stacked `<section className="panel">` blocks in `TrendsView` (matches `EosView` `flex flex-col gap-6`) |
| xl | 32px | `p-8` `TrendsView` `<main>` outer padding (matches `EosView`/`HostsView` precedent) |
| 2xl | 48px | not used in this phase |
| 3xl | 64px | not used in this phase |

Exceptions:
- **Sparkline fixed height = 36px** (`style={{ height: 36 }}` on `<Chart>`). 36 is a deliberate row-height exception for the dashboard cluster-card sparkline slot (RESEARCH Pattern 3); it is a chart canvas dimension, not a layout-spacing token, and is the single non-4-multiple value in scope. Documented and intentional.
- **Trend line chart min-height = 320px** — inherits the shipped `<Chart>` default (`style ?? { minHeight: 320 }`, Chart.tsx:101). No override.

## Typography

vatlas has no custom type scale; it composes Tailwind defaults already established across P2–P7. Phase 8 reuses the exact roles shipped in `EosView`/`SnapshotCard` — no new sizes, no new weights.

| Role | Size | Weight | Line Height | Phase 8 usage |
|------|------|--------|-------------|---------------|
| Display (metric value) | 24px (`text-2xl`) | 600 (`font-semibold`) via mono `tabular-nums` | Tailwind default (1.0–1.2 for 2xl) | Delta-panel primary numbers, headline KPI values (mirrors `EosView` tile `text-2xl tabular-nums`) |
| Heading | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 (Tailwind `text-xl` default) | `TrendsView` `<h2>` section heading (verbatim `EosView` `text-xl font-semibold`) |
| Subheading | 16px (`text-base`) | 600 (`font-semibold`) | 1.2 | Delta-panel / chart sub-section `<h3>` (verbatim `EosView` `text-base font-semibold`) |
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 (Tailwind `text-sm` default ≈1.43, treated as body) | Factual captions: warm-up "trends preparing — N/M", inferred-order note, released-snapshot note, error state |
| Caption / meta | 12px (`text-xs`) / `text-[12px]` | 400 (regular) | default | Snapshot-card metadata, per-point tooltip body, empty-state text (verbatim `SnapshotCard` / `SnapshotListSidebar` precedent) |

Declared scale: **4 sizes** (24 / 20 / 16 / 14, with 12 as the existing meta/caption tier shipped project-wide) · **2 weights** (regular 400 + semibold 600). No italic, no third weight. Numerics always `font-mono tabular-nums` (project convention, CLAUDE.md / `EosView` precedent) so trend values and deltas do not jitter as they update progressively.

## Color

Phase 8 introduces **no new color**. It consumes the shipped Midnight Executive `@theme` tokens (`src/index.css`) exactly as P2–P7 do. Hex approximations below are the documented oklch anchors from `index.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (~60%) | light `#FAFAFA` `bg-slate-50` / dark `surface-900` (`oklch(15% .02 260)`) | App background, `<main>` canvas behind the trends view |
| Secondary (~30%) | light `#FFFFFF` `.panel` (`bg-white`) / dark `surface-800` (`oklch(20% .02 260)`); sidebar border `slate-200`/`surface-700` | Trend chart panel, delta-panel card, snapshot-list rail, dashboard cluster cards hosting sparklines |
| Accent (~10%) | gold `--color-accent-500` `oklch(78% .16 75)` ≈ `#F9B935` | **Reserved — see explicit list below** |
| Destructive | `--color-util-high` `oklch(58% .22 25)` (red) | **Hover-only** on the snapshot-card remove `✕` (`hover:text-util-high`, shipped `SnapshotCard` idiom). No destructive *confirmation* surface in this phase (see Copywriting). |

**Accent gold reserved for (exhaustive — never "all interactive"):**
1. The **active segment** of the `ViewToggle` (the shipped contract: `bg-accent-500 text-surface-900` on the selected view segment — the new `'trends'` segment follows this verbatim, no new style).
2. The **active/selected snapshot card** ring in the sidebar (`ring-2 ring-accent-500 dark:ring-accent-400`, shipped `SnapshotCard` idiom — unchanged; the active snapshot is the D-01 latest-`capturedAt` estate).

Accent is **not** applied to: trend line series, sparkline strokes, delta numbers, the warm-up indicator, the inferred-order caption, or tooltips. Those use neutral series/text tokens.

**Chart color contract (TRD-02/03/04 — no verdict, no traffic-light):**
- Trend line + sparkline strokes use the **Midnight Executive theme series palette** already registered on `<Chart>` (`echarts.registerTheme('midnight-executive' | '…-dark')`, Chart.tsx:45–46). No per-series semantic color, no green/orange/red on trends — a metric going up or down carries **no judgment** (PROJECT.md no-editorial-verb / no-verdict-color invariant; P7 D-00 factual-presentation precedent applies to trends and deltas).
- `--color-util-low/mid/high` (the status ramp) is **forbidden** on any trend/delta/sparkline element — those tokens are utilization-status only and would imply a verdict on direction of change.
- Delta sign is communicated by the **text glyph** (`+` / `−` / em-dash sentinel `—`), never by color. `+12` and `−3` are the same neutral text color.
- Sparkline stroke: single neutral series color, `lineStyle:{ width: 1.5 }`, `showSymbol:false` (RESEARCH Pattern 3). Each cluster card's sparkline uses the **same** stroke token (not a per-cluster color) — it is a shape, not a categorical legend.

Every color utility MUST carry its `dark:` twin (CLAUDE.md class-strategy dark mode — a missing pair renders invisibly in one theme; this is a hard gate on every new className in scope).

## Copywriting Contract

All Phase-8 strings are **factual, no editorial verb** (PROJECT.md denylist: no "grew / shrank / improved / degraded / should / recommend / poor / good"), carry **no pre-formatted numbers** (numbers injected via `utils/format.ts` `fmtDate`/`fmtInt`/`fmtMemMb` interpolation, never baked into the string), and ship as **EN + FR parity keys** in a new `i18n/locales/{en,fr}/trends.json` namespace (mirrors the shipped `eos.json` precedent). The exact key strings are the planner's call within this contract; the table below fixes intent + tone.

| Element | Copy intent (factual, no verb, EN/FR parity) |
|---------|----------------------------------------------|
| Primary CTA / entry | **No new CTA.** Trends is reached via the existing `ViewToggle` segment labelled `trends` (`nav.trends` key, added to the shipped `inventory:nav.*` set). The view *exists* when ≥2 snapshots are loaded; the act that "enters" trends is the existing UploadZone drop + the existing view segment. |
| Empty state heading | A neutral heading equal to the section title (precedent: `EosView` renders just the `<h2>` heading when no snapshot). E.g. the `trends:heading` string ("Trends" / "Tendances"). |
| Empty state body | Factual single line stating the prerequisite: trends require ≥2 loaded snapshots (e.g. "Load 2 or more snapshots to see trends." / "Chargez 2 instantanés ou plus pour voir les tendances."). One snapshot ⇒ `view.trends` is `null` (RESEARCH Pattern 1); render this line, no chart, no spinner. State the condition, not advice. |
| Warm-up indicator (D-02) | Factual progress line scoped to the snapshot-list/trends area only (never a global blocking overlay): "trends preparing — N/M" with N=`done`, M=`total` injected via `fmtInt` (e.g. `trends:warmup` = `Trends preparing — {{done}}/{{total}}` / `Tendances en préparation — {{done}}/{{total}}`). No spinner-as-blocker, no "please wait", no verb. Disappears when `done === total`. Lives in `SnapshotListSidebar`'s own region. |
| Inferred-order caption (D-05) | Neutral fact that ordering is not from real capture dates, shown only when `view.trends.orderInferred === true`: e.g. `trends:orderInferred` = "Timeline order inferred — not from real capture dates." / "Ordre de la chronologie déduit — non issu des dates de capture réelles." No "warning", no color, no icon — a plain `text-sm text-slate-500 dark:text-slate-400` caption (the shipped `EosView` caption tone). |
| Released-snapshot note (DD-C) | Shown when a released snapshot (`rawReleased === true`) is clicked / cannot become the active estate: factual, e.g. `trends:released` = "Raw data released to free memory — trend point retained." / "Données brutes libérées pour économiser la mémoire — point de tendance conservé." No verb, no alarm; selection of a released snapshot for the active merge is prevented or factually explained, not error-styled. |
| Delta-panel rows (TRD-04) | One factual row per consecutive pair, per metric: a label key + a signed branded number, e.g. label `trends:delta.vmCount` ("VMs") rendered as `+12` / `−3`; memory/storage in branded GiB/TiB via `fmtMemMb` (never `* 1.048576`). Sign `+`/`−` is factual; em-dash `—` when the field is `null` on either side of the pair. No "increase/decrease", no color, no arrow-up/down glyph beyond the mathematical sign. |
| Per-point tooltip (criterion 6) | ECharts `tooltip:{trigger:'axis'}` body lists, factually: the point date (locale via `fmtDate`) and the contributing snapshot metadata array — each `{ vCenterLabel, rvtoolsVersion }` (A2: a merged point legitimately spans multiple vCenters). Plain key/value, no narrative. |
| Error state | Region-scoped `ErrorBoundary` fallback identical to `EosView`'s `EosError`: `role="alert"`, reads **only** `error.message` (never `error.cause`/`stack`/the object — would leak VM/host names), rendered in a `border-util-high/40` panel. Copy = the shipped `dashboard:states.error` pattern (`{{message}}` interpolated). No retry button (no async user action to retry — refresh is the recovery, consistent with the privacy/in-memory model). |
| Destructive confirmation | **None in this phase.** The only destructive-adjacent action is the existing per-snapshot remove `✕` (shipped `SnapshotCard`, hover-`util-high`, **no confirmation dialog** by established product pattern — DRX-01 reversible/neutral precedent). Memory release (DD-C) is automatic and non-destructive to the trend series (the point is retained); it requires no user confirmation and no destructive styling. |

## Component & Interaction Contract (Phase-8 specifics)

> This section binds the executor to shipped idioms. It is not in the base template but is required for an accurate trends contract.

### View navigation — extend, do not invent
- Add `'trends'` to `AppView` and `VIEWS` in the shipped `ViewToggle.tsx`. **Do not create a new nav component.** The `<fieldset role="group">` + `<legend className="sr-only">` + `map(button aria-pressed)` idiom, the gold-active-segment style, the Arrow-Left/Right wraparound keyboard handler, and `focus-visible:ring-2 ring-primary-500` are inherited verbatim (ViewToggle.tsx:47–75). New label key `inventory:nav.trends` (EN + FR).
- Segment ordering is the planner's call; recommend appending `trends` last (after `eos`) to preserve muscle memory for shipped segments.

### Trend chart (TRD-02) — `TrendChart.tsx` → shipped `<Chart>`
- Renders through the **single shipped `<Chart>` primitive** (`option` prop only). No new chart component, no renderer change (SVG is injected by `<Chart>`, VIZ-01).
- `xAxis:{ type:'time' }` + `series.data` as `[Date, value]` pairs — **mandatory**; `type:'category'` is forbidden as the default (Pitfall 1 / criterion 2: non-uniform real-date spacing must be visible for irregular dates e.g. 2026-01-31, 2026-02-15, 2026-03-30).
- D-05 presentation rule (locks Open Question 1, recommendation (a)): when `view.trends.orderInferred === true`, the chart renders with `xAxis:{ type:'category' }` over the stable-ordered labels **plus** the inferred-order caption; when all points carry real distinct dates, `xAxis:{ type:'time' }`. Whole-series, never mixed — honest, no fabricated positions. Data layer always keeps real dates where known (D-05).
- `axisLabel.formatter` defers to `fmtDate`/locale — never a baked formatted string in i18n (FR/EN parity).
- `tooltip:{ trigger:'axis' }` carries the per-point metadata (criterion 6).
- `LineChart` is **not yet registered** in `Chart.tsx`'s `echarts.use([...])` (only Bar/Pie/Gauge/Heatmap). This UI-SPEC states the line/sparkline visual contract; **the planner adds `import { LineChart } from 'echarts/charts'` + registration and re-runs `npm run check:bundle-size` (≤300 KB gz gate)** (RESEARCH gap A1 / Pitfall 2). The visual contract here does not change regardless of registration mechanics.

### Per-cluster sparkline (TRD-03) — `TrendSparkline.tsx` → same `<Chart>`
- Same `<Chart>` component, `type:'line'` option with axes hidden (`xAxis/yAxis show:false`), grid collapsed full-bleed (`grid:{top:2,bottom:2,left:2,right:2}`), `showSymbol:false`, `tooltip` off, `lineStyle:{width:1.5}`, `style={{ height: 36 }}` (RESEARCH Pattern 3). One neutral stroke token, no per-cluster color.
- Slot lives in the shipped dashboard cluster card (`dashboard/ClusterColumn.tsx`); renders **only when `view.trends` has ≥2 points** for that cluster. With <2 points the slot is absent (no placeholder box, no skeleton — matches the empty-state "state the condition" tone). Sparklines fill progressively as warm-up lands (D-02, store→memo reactive path; no per-sparkline spinner).

### Snapshot list — inline capture-date edit (D-03) + warm-up + caption
- `SnapshotCard.tsx` gains an **inline-editable `capturedAt`** control replacing the read-only `snapshot.capturedAt.toLocaleDateString()` text. Interaction contract: a date input that on commit calls the **shipped `setCapturedAt` store action** (no new mutation, no local date state that bypasses the store — Pitfall 5). The edited value is the highest-priority capture-date source (D-03). Editing recomputes the series through the single memo automatically. Styling: stays within the existing `text-xs` card-meta type tier; the input inherits panel/dark-twin tokens; focus ring `focus-visible:ring-2 ring-primary-500` (project focus idiom). No accent gold on the input (accent is reserved — see Color).
- Warm-up "trends preparing — N/M" line: rendered in `SnapshotListSidebar`'s own region (scoped, never global), `text-sm text-slate-500 dark:text-slate-400`, present only while `done < total`. State source = `useSnapshotUpload` exposing `{ done, total }`.
- Inferred-order caption: rendered once near the trend chart and/or snapshot list when `orderInferred`, neutral `text-sm` slate caption.

### Delta panel (TRD-04) — `DeltaPanel.tsx`
- A `<section className="panel">` of factual consecutive-pair rows (DD-B B1: count-deltas on existing aggregate fields — vmCount, poweredOnVms, hostCount, clusterCount, vcpuAllocated, vramAllocated→GiB, totalStorage→GiB/TiB; final visible set is the planner's pick within DD-B). Each row: label (i18n key) + signed branded number (`font-mono tabular-nums`), `—` em-dash when null on either side. No color, no traffic-light, no up/down arrow glyph, no verb. Layout mirrors the `EosView` reconcile/split caption tone for secondary factual lines.

### Layout shell — `TrendsView.tsx`
- `<main className="flex-1 overflow-y-auto p-8">` wrapping an `ErrorBoundary` + `<div className="flex flex-col gap-6">` of `<section className="panel">` blocks — **verbatim `EosView` shell** (heading section → chart section → delta section). No new layout primitive.

### Accessibility (inherited contract)
- All toggles/segments: `aria-pressed` + visible `focus-visible:ring-2 ring-primary-500` (project idiom — CLAUDE.md ThemeToggle/ViewToggle precedent).
- Charts: pass an `ariaLabel` to `<Chart>` (the `aria-label` hook, Chart.tsx:62) describing the metric/series factually.
- Error fallback: `role="alert"`.
- New i18n keys (`trends:*`, `inventory:nav.trends`) MUST exist in BOTH `en/` and `fr/` (CLAUDE.md i18n gate) with no pre-formatted numbers.
